# LLM-judge 軸③ 保護者向け表現の適切さ

設計コンテキスト §12 の採点3軸の3つ目。生成された書類の表現が保護者・現場にとって適切か
（断定的な発達評価を避ける／観察事実と解釈の分離／月齢・発達段階に照らした自然さ／個人名を書かない）
を 0–1 で採点する。

## 採点プロンプト

あなたは保育現場の文章を点検する評価者です。次のドラフトの表現を 0.0–1.0 で採点してください。

入力:
- draft: {{draft}}

採点基準:
1. **実名を書いていない**（子ども・保護者は仮名/属性で表す＝架空児のみ・§14）。
2. **断定的な発達評価を避ける**（「できない」「遅れている」等の決めつけでなく、観察に基づく記述）。
3. **観察事実と解釈の分離**（見たこと と それをどう捉えたか が混ざっていない）。
4. **保護者に伝わりやすい**自然な表現で、月齢・発達段階に照らして無理がない。

スコアの目安:
- 1.0：実名なし・事実/解釈が分離・断定を避け・自然で伝わりやすい。
- 0.5：おおむね良いが、解釈と事実がやや混在／一部に硬さや決めつけ気味の表現。
- 0.0：実名記載、または断定的な発達評価が中心。

must_fix:
- 子ども/保護者の**実名**が書かれている（共通ルール違反＝§14）。
- 「発達が遅れている」等の**断定的な発達評価**。

出力（JSON のみ）:
{"score": 0.0-1.0, "rationale": "根拠", "must_fix": ["違反があれば列挙、なければ空配列"]}
